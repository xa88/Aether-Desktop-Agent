# Aether Desktop Agent (ADA) - Enterprise Deployment Guide

This manifesto dictates standard operating procedures for zero-touch configuration and mass deployment of the ADA Client binaries utilizing configuration management solutions like Intune or SCCM.

## 1. Zero-Touch Client Deployment (Windows)
The ADA package exposes standard `MSIExec` parameters. Administrators must supply MSI flags to configure background orchestration services statically out of LLM bounds.

**Headless Endpoint Installation:**
```bash
msiexec /i ADAInstaller.msi /qn /norestart ALLUSERS=1 INSTALLDIR="C:\Program Files\AetherAgent\"
```

**Injecting Global Policies (Offline/Airgapped Flags):**
To guarantee an offline endpoint upon boot, statically construct the Environment variable globally pointing to a certified `profile.yaml`:
```bash
msiexec /i ADAInstaller.msi /qn ADA_PROFILE_URI="C:\ProgramData\ADA\profile.yaml" CLOUD_ENABLED=false
```

## 2. Setting Up Remote Sandboxes (DMZ Clusters)
Instead of permitting executing tools laterally on Developer workstations directly, ADA allows pushing workloads natively through `ada-sandbox` to dedicated HTTP clusters.

1. Deploy the `ada-worker` linux binary onto an Ubuntu KVM or EKS node.
2. Ensure TLS via Nginx / Envoy using valid corporate certificates.
3. Supply developers the URI override inside `profile.yaml`:
```yaml
remote_sandbox: "https://ada-dmz-worker.corp.local:3000"
```

## 3. Mandatory Drift Checking (Drift CI)
Configuration drifts on the Electron interface represent severe XSS exposures. Configure an explicit CRON on developer endpoints validating the immutable hash generated by `desktop/scripts/drift_check.js`. If modified without an administrative override, immediately kill the Node.js root process tree.
